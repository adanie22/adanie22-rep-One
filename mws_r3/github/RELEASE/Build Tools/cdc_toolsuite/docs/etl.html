Export, Transform & Load¶

Workflow¶

Designed for batch operations, currently run from test routines in test_etlxx.

Export, Transform, Load workflow is:

1) Export: take trunk or branch from Code Repository, or export directly out of Agility using REST API into local Working Copy (WC).

  1. Populate cdc_CIs.json for environment containers and stacks for new environment
  2. Transform WC using cdc_CIs.json as input.
  3. Check changes back into branch with tag.
  4. Import asset XML into new environment.

Report Parameter File: cdc_CIs_json¶

To support batch running, reports, extraction routines, use configuration file cdc_CIs.json, of below format:

{
    "blueprints": {
        "143a": {"CID":1,"SCID":2,"BASELINE":["PRY_R3"]},
        "1431": {"CID":"TEST","SCID":"DH","BASELINE":["PRY_R3"]},
        "330":  {"CID":"MAS","SCID":"FARM","BASELINE":["PRY_R3"]}
    },..
}

A selection of test routines using cdc_CIs:

test.test_etl.export_all1_test_runner()

test.test_report.xls_s_report1()

The consumer of a list of ID’s gathered from above will be eg, the get_assets function

cdc.blueprints.get_assets()
